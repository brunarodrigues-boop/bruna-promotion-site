Delivered

Platform
TimeBack Dashboard — built from a single commit to a production platform
FastAPI + React 19 + TypeScript + Tailwind v4, deployed on Railway via Docker with CI/CD through GitHub Actions. Tracks XP, time, accuracy, tests, and grade-level progress per student per subject across 12+ schools. Role-based access covers guide views, admin views, cross-school composite roles, and doom viewer roles. Fourteen report endpoints cover full student reports, daily alerts, intervention PDFs, weekly summaries, and MAP results. Built features include Grade Progress History, a Notes tab, a Messaging tab with Slack DM and PDF preview-and-send, a CSV export endpoint, and a public Reporting API with full documentation. Every feature is in daily use.
Summer Camp
Miami Summer Camp — zero to production
Built end-to-end with no prior infrastructure, in parallel with ongoing operations across all other schools. Alpha had nothing like this before — a week picker for guides, a Test Tracker with live Google Sheets integration, and bulk ZIP download of all guide XP reports. Intervention detection was scoped to the summer period specifically to avoid false signals from the prior school year.
Summer Camp
Public School Summer Camp — 508 students, zero prior infrastructure
Onboarded a cohort of 508 public school students — the largest single cohort ever tracked on the platform. Extended the Miami Summer Camp infrastructure to a completely separate school with its own data mapping, CSV-based access, and reporting pipeline.
Onboarding
School onboarding — 12+ campuses, every one live
Alpha Miami, Santa Barbara, Palm Beach, Fort Worth, NextGen Academy, Waypoint Academy, Austin (virtual school split architecture for L1/L2/WL+LL), Brownsville, Immaculate Academy, ScienceSIS, Vita High School, and a public school summer camp cohort. Each onboarding required custom data mapping, name variant resolution across OneRoster, guide assignment configuration, and data quality investigation. Every school has distinct edge cases; every school is live.
Reporting
Custom PDF pipelines — 600+ reports, school-wide and per-student
End-of-year student reports generated individually per school, each containing MAP data, STAAR standards classification, and a mastered test breakdown — pulled directly from the OneRoster API. Gap analysis PDFs produced using Claude: wrong answers from failed tests are read, skill gaps identified, and actionable coaching recommendations formatted per student per subject. A weekly pipeline produces 600+ PDFs per run.

How I work

01
Data quality runs as a continuous process, not a one-time fix. 25+ OneRoster name variants documented and resolved across schools. MAP data reconciled across Fall and Winter cycles. Cross-enrollment and missing guide assignments detected and flagged. Every fix is a commit; nothing falls through.
03

05
AI integration across every workflow.
